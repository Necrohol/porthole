About Porthole

What is Porthole

Porthole is a GTK-based frontend for Portage (the Gentoo package management system, http://www.gentoo.org).  It features a hierarchical view of packages and supports fast searches and dependency calculations.  Porthole also supports basic emerge features such as emerge, unmerge, sync and upgrade world.  Porthole is written in Python.

History

Porthole is partially based on an earlier program, called gportage ( http://www.stacken.kth.se/~foo/gentoo/), which has been discontinued in favour of Porthole.

For a detailed history of changes, please see NEWS.

How to Install Porthole

The easiest and recommended way of installing Porthole is by using the ebuild that is available from the webpage.  We assume you know how to put ebuilds in the portage overlay.  If not, please consult the documentation and forums on the Gentoo web site (http://www.gentoo.org).

If you don't want to use the ebuild, you can download the tarball (porthole-0.3.1.tar.bz2) and install it yourself.  The following packages must be installed for Porthole to work:

  • >=python-2.3
  • >=PyGTK-2 and libglade-2
  • >=pyxml-0.8.2
  • and, of course, Portage

Use the following commands to install Porthole: 

> tar xvjf porthole-0.3.1.tar.bz2
> cd porthole-0.3.1
> python setup.py install

How to Run Porthole

From a terminal window enter: 

> porthole

Porthole has only two possible command line flags:

  • --debug (this generates extra diagnostic info)
  • --local (used for a non-emerged copy of Porthole)

For Gnome users, Porthole will install a menu item under Applications -> System Tools.

Tip! If you run into a problem, try running porthole --debug from a terminal.
